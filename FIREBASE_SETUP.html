🔥 Firebaseセットアップ手順書

ドライバルク チャータリング — マルチプレイ版を動かすための設定ガイド（所要時間：約15〜20分）

このガイドでやること： GoogleのFirebaseというサービスに無料アカウントを作り、マルチプレイのデータを保存する「データベース」を作ります。プログラミングの知識は不要です。

1
Googleアカウントでサインイン

Firebaseは Google が提供する無料サービスです。Gmailアカウントがあればすぐ使えます。

  1. ブラウザで https://console.firebase.google.com を開く
  2. Googleアカウントでログインする（Gmailと同じアカウント）
  3. Firebaseコンソールのトップページが表示されればOK
🖼️ 画面上部に「Firebaseへようこそ」のような文字が表示されるはずです

2
新しいプロジェクトを作成

  1. 「プロジェクトを作成」（または「＋ プロジェクトを追加」）をクリック
  2. プロジェクト名を入力（例：chartering-game）— 半角英数字・ハイフンのみ
  3. 「続行」をクリック
  4. 「このプロジェクトでGoogle アナリティクスを有効にする」は オフ でOK（チェックを外す）
  5. 「プロジェクトを作成」をクリック
  6. しばらく待つと「新しいプロジェクトの準備ができました」と表示 → 「続行」
🖼️ プロジェクト名は後で変更できます。自分がわかればなんでもOKです

3
Realtime Database を作成

マルチプレイのデータ（ルーム情報・市場データ・プレイヤー情報）を保存するデータベースを作ります。

  1. 左サイドバーの「構築」→「Realtime Database」をクリック
  2. 「データベースを作成」をクリック
  3. ロケーション選択：「asia-southeast1（シンガポール）」 を選ぶ（日本から近くて速い）
  4. 「次へ」をクリック
  5. セキュリティルール：「テストモードで開始」 を選択
  6. 「有効にする」をクリック
⚠️ テストモードについて： テストモードは「誰でも読み書き可能」な設定です。友達と遊ぶ分には問題ありませんが、30日後にルールが自動的に変わります。本格的に公開する場合は後でルールを設定してください。

4
アプリを登録してAPIキーを取得

ゲームのHTMLファイルがFirebaseに接続するための「鍵（APIキー）」を取得します。

  1. プロジェクトの概要ページに戻る（左上のプロジェクト名をクリック）
  2. 「アプリを追加」の下にある 「</>」（ウェブ）アイコン をクリック
  3. アプリのニックネームを入力（例：chartering-web）
  4. 「Firebase Hosting も設定する」は チェックしない
  5. 「アプリを登録」をクリック
  6. 以下のようなコードが表示される — この設定値をすべてコピーしておく
// コピーする設定値（値は例です。あなたの画面に表示される実際の値を使ってください） const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "chartering-game-xxxxx.firebaseapp.com", databaseURL: "https://chartering-game-xxxxx-default-rtdb.asia-southeast1.firebasedatabase.app", projectId: "chartering-game-xxxxx", storageBucket: "chartering-game-xxxxx.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef1234567890" };
💡 コピーのコツ：「コピー」ボタン（または Ctrl+A → Ctrl+C）でコード全体をコピーし、メモ帳などに一時保存しておくと便利です。

5
multiplayer.html に設定を貼り付け

コピーした設定を multiplayer.html ファイルに貼り付けます。

  1. テキストエディタ（メモ帳、VSCode など）で multiplayer.html を開く
  2. ファイルの先頭付近に以下のような箇所を探す：
// ▼▼▼ Firebase コンソールからコピーした設定を貼り付け ▼▼▼ const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT_ID.firebaseapp.com", databaseURL: "https://YOUR_PROJECT_ID-default-rtdb.firebaseio.com", projectId: "YOUR_PROJECT_ID", storageBucket: "YOUR_PROJECT_ID.appspot.com", messagingSenderId: "YOUR_SENDER_ID", appId: "YOUR_APP_ID" }; // ▲▲▲ ここまで ▲▲▲
  1. YOUR_API_KEY などの YOUR_〜 の部分を、Firebaseからコピーした実際の値に置き換える
  2. ファイルを保存する（Ctrl+S）
⚠️ 注意： databaseURL の値が特に重要です。Realtime Databaseを「asia-southeast1」で作った場合、URLに asia-southeast1.firebasedatabase.app が含まれます。コンソールに表示された値をそのままコピーしてください。

6
GitHub Pages にアップロードして完成

設定済みの multiplayer.html をウェブ上に公開します。

  1. 既存の chartering-game リポジトリ（ソロプレイ版）を GitHub で開く
  2. 「Add file」→「Upload files」をクリック
  3. 設定済みの multiplayer.html をドラッグ＆ドロップ
  4. 「Commit changes」をクリック
  5. 数分後に以下のURLでアクセス可能：
https://あなたのユーザー名.github.io/chartering-game/multiplayer.html
💡 このURLを友達に共有すれば、マルチプレイが楽しめます！ソロプレイ版（index.html）とは別ファイルなので、両方そのまま使えます。

✓
動作確認

  1. URLをブラウザで開く → ロビー画面が表示される
  2. プレイヤー名を入力 → 「新しいルームを作る」をクリック
  3. 4文字のルームコードが表示される
  4. 別のデバイスやブラウザで同じURLを開き、ルームコードを入力して入室
  5. ホストが「ゲームスタート」を押すと全員同時開始
🖼️ うまくいかない場合：ブラウザのコンソール（F12キー）でエラーメッセージを確認してください

❓
よくあるトラブルと対処法

  • 「ルームが見つかりません」と出る： ルームコードが正しいか確認。大文字・小文字は自動変換されます。
  • ゲームが始まらない： Firebase コンソールの Realtime Database で「データ」タブに rooms/ のデータが増えているか確認。増えていなければAPIキーが誤っています。
  • 「PERMISSION_DENIED」エラー： テストモードの30日期限が切れた可能性があります。Firebase コンソール → Realtime Database → ルール → 以下に変更して「公開」:
{ "rules": { ".read": true, ".write": true } }
  • 画面が更新されない： ブラウザをハードリロード（Ctrl+Shift+R または Cmd+Shift+R）してください。

🎉 セットアップ完了！
